## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (Tailwind: `gap-1`, `px-1`) |
| sm | 8px | Compact element spacing (Tailwind: `gap-2`, `p-2`) |
| md | 16px | Default element spacing (Tailwind: `gap-4`, `p-4`) |
| lg | 24px | Section padding (Tailwind: `gap-6`, `p-6`) |
| xl | 32px | Layout gaps (Tailwind: `gap-8`, `p-8`) |
| 2xl | 48px | Major section breaks (Tailwind: `gap-12`, `p-12`) |
| 3xl | 64px | Page-level spacing (Tailwind: `gap-16`, `p-16`) |

**Exceptions:**
- Sidebar width: 48px (w-12) — established in Phase 1
- Message bubbles: 16px vertical padding, 24px horizontal padding (comfortable reading area)
- Conversation list default width: 280px (resizable 20%-40% of viewport)

**Chat-specific spacing:**
- Message list scroll padding: 16px bottom (keeps last message above input field)
- Conversation list item gap: 4px (dense list, matches Linear/Raycast)
- Code block internal padding: 16px (allows copy button positioning)
- File attachment preview gap: 8px (multiple files in row)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |
| Heading (conversation title) | 14px | 500 (medium) | 1.4 |
| Code (monospace) | 13px | 400 (regular) | 1.6 |

**Notes:**
- Base font size: 14px (set via `--font-scale: 1` in `src/index.css`)
- Body text weight 400 is primary reading weight for message content
- Weight 500 used for emphasis: conversation titles, labels, model badges
- Code blocks use 13px (1px smaller than body for density) with 1.6 line-height (readability for long code)
- Markdown headings within messages: inherit body weight (500), use size scale (16px/18px/20px) for h1/h2/h3

**Font stack:** Inherited from Phase 1 (system font stack defined in Tailwind v4 defaults)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(224 71.4% 4.1%)` (dark mode) | Main background, message list background |
| Secondary (30%) | `hsl(215 27.9% 16.9%)` (dark mode) | Conversation list background, user message bubbles, code block backgrounds |
| Accent (10%) | `hsl(263.4 70% 50.4%)` (violet, default) | **Reserved for:** Active conversation ring, send button, model badge (current), stop streaming button, focus rings |
| Destructive | `hsl(0 72.2% 50.6%)` (red) | Delete conversation action only |

**Accent reserved for:**
1. Active conversation indicator (2px ring on selected conversation)
2. Send button background (primary CTA)
3. Stop streaming button (interrupt action)
4. Model badge for current model in message header
5. Focus rings (keyboard navigation)

**NOT used for:** Message text, links within markdown, timestamps, generic interactive elements

**Color semantics:**
- User messages: `bg-secondary` (slightly elevated from background)
- AI messages: `bg-card` (same as dominant background, but distinct container)
- Code blocks: `bg-muted` (slightly lighter than secondary for contrast)
- Inline code: `bg-muted` with `text-muted-foreground`

**Light mode:** Phase 1 established light mode tokens (not dark-first, but supported). Phase 2 inherits light mode support but optimizes for dark mode.

## Interaction Patterns

### Streaming Chat Flow
1. User types message in auto-growing textarea (bottom-fixed)
2. User clicks "Enviar" or presses Enter (Shift+Enter = newline)
3. User message appears immediately in chat (optimistic UI)
4. Typing indicator (3 dots) appears in AI message bubble
5. Tokens stream in real-time, markdown renders incrementally
6. Auto-scroll follows if user is near bottom (within 100px)
7. Stop button appears during streaming (cancels request)
8. On completion, Regenerate button appears on AI message

### File Attachment Flow
1. User clicks paperclip icon OR drags file over chat area
2. File picker opens (filter: PDF, images, txt/md/docx)
3. Selected file shows preview (image thumbnail OR doc icon + filename)
4. User can remove file before sending (X button on preview)
5. On send, file encodes as base64 and sends with message
6. AI message references file content

### Model Switching Flow
1. Model picker shows current model as button label ("GPT-4o ▼")
2. User clicks to open dropdown (list of configured models from settings)
3. User selects new model
4. Next message uses new model
5. Message header shows model badge (which model answered)

### Conversation Management Flow
1. User clicks "Nova Conversa" — creates new conversation, focuses input
2. Conversation list shows "Nova Conversa — [timestamp]" initially
3. After first AI response, title auto-generates (LLM call, 3-8 words)
4. User can right-click conversation → "Excluir" → confirmation dialog
5. Deleted conversation soft-deletes (SQLite `deletedAt` column)

### Keyboard Shortcuts
- `Enter` — Send message
- `Shift+Enter` — Newline in input
- `Cmd/Ctrl+K` — New conversation
- `↑/↓` — Navigate conversation list (when focused)

## States

### Message States
- **Sending (user message):** Optimistic UI, message appears immediately, subtle opacity until confirmed
- **Streaming (AI message):** Typing indicator → tokens appear → stop button visible
- **Complete (AI message):** Regenerate button appears on hover
- **Error (API failure):** Error message inline with retry button

### Conversation List States
- **Empty (no conversations):** Centered empty state in list area
- **Searching:** Real-time filter, no results state: "Nenhuma conversa encontrada"
- **Loading:** Skeleton items while fetching from DB (first load only)

### Input States
- **Idle:** Placeholder: "Digite sua mensagem..."
- **Typing:** Auto-grow textarea (max 200px height, then scroll)
- **Streaming:** Input disabled, "Parar" button replaces "Enviar"

### Attachment States
- **No attachment:** Paperclip icon only
- **File selected:** Preview appears above input, X button to remove
- **Too large (>10MB):** Error toast: "Arquivo muito grande. Limite: 10MB"
- **Unsupported type:** Error toast: "Tipo de arquivo não suportado"

---

## Accessibility

- **Keyboard navigation:** All interactive elements focusable, visible focus rings (ring-2 ring-ring)
- **Screen reader support:** Semantic HTML (`<nav>`, `<article>` for messages), ARIA labels on icon-only buttons
- **Color contrast:** All text passes WCAG AA (4.5:1 for body text, 3:1 for large text)
- **Focus management:** On "Nova Conversa", focus moves to message input
- **Loading states:** Aria-live regions for streaming messages (screen reader announces new tokens)

---

## Responsive Behavior

Phase 2 is desktop-only (Windows + macOS via Tauri). No mobile breakpoints required.

**Minimum window width:** 800px (sidebar 48px + conversation list 280px + chat view 472px minimum)
**Below minimum:** Conversation list auto-collapses to icon-only (future enhancement, not Phase 2)

---

## Animation

- **Message appearance:** Fade-in 150ms ease-out
- **Typing indicator:** Pulse animation (3 dots, 600ms cycle)
- **Auto-scroll:** Smooth scroll behavior (150ms duration)
- **Resizable panel:** No animation (immediate resize, localStorage persists)
- **Copy button state change:** 200ms fade transition ("Copiar" → "Copiado!")
