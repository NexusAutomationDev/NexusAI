    Create src-tauri/crates/nexusai-kb/src/store.rs — the rusqlite-side persistence using a shared KB connection:
    - A lazy/once `pub fn kb_connection(db_path: &Path) -> Result<...>` opening the same `nexusai.db` file used by tauri-plugin-sql, with `PRAGMA foreign_keys=ON`. On first open, call `vector::register_sqlite_vec()` (must run before opening, so register at app startup in Task 3 instead — document the ordering), `vector::init_vec_table(&conn)`, and `search::init_fts_table(&conn)`.
    - `pub fn index_item(conn, item_id: &str, text: &str, model: &TextEmbedding, ch: &Channel<IndexProgress>) -> anyhow::Result<()>`:
      1. emit `Started { item_id, total_chunks: None }` then chunk via `chunk::chunk_default(text)`.
      2. emit `Started { item_id, total_chunks: Some(n) }` (or a `Chunk` progress per batch) so the UI can show "Indexando X de Y" (D-11).
      3. `embed::embed_passages(model, &chunk_texts)`.
      4. In a transaction: INSERT each kb_chunks row (id = uuid, item_id, ordinal, content, section, char_start/end) returning the integer rowid; `vector::insert_vector(conn, rowid, &emb)`; FTS stays in sync via the triggers from Plan 03-02.
      5. UPDATE kb_items SET status='indexed', updated_at=now WHERE id=item_id; emit `Indexed { item_id }`.
      On any error: UPDATE kb_items SET status='failed', error_reason=<msg> and emit `Failed { item_id, reason }` (D-12). Wrap so a failure is terminal + reconcilable.
    - `pub fn reindex_item(conn, item_id, ...)`: idempotent — `DELETE FROM kb_chunks WHERE item_id=?` (collect their rowids first), `vector::delete_vectors_for_rowids(conn, &rowids)`, then re-run index_item (D-12, must be inside a transaction).
    - `pub fn query_chunks(conn, fused_rowids: &[i64]) -> Result<Vec<Citation>>`: SELECT kb_chunks JOIN kb_items to build Citation { id, item_id, item_title, kind, section, snippet=content }. NO agent/owner filter — single shared index (KB-06).
