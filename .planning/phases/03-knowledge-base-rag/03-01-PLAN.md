    Implement `pub fn chunk_text(text: &str, target_tokens: usize, overlap_tokens: usize) -> Vec<Chunk>` in chunk.rs:

    - Token approximation: use a chars-per-token heuristic of ~4 chars/token (so target ~512 tokens ≈ ~2048 chars). Define `target_chars = target_tokens * 4` and `overlap_chars = overlap_tokens * 4`.
    - First pass — segment the text into blocks: split on Markdown headings (lines starting with `#` ... `######`) and on blank-line paragraph boundaries (`\n\n`). Track the current `section`: when a heading line is encountered, set `section = Some(heading_text_without_#)` for subsequent blocks until the next heading.
    - Second pass — pack consecutive blocks into chunks up to `target_chars`. When a chunk closes, start the next chunk by including an overlap window of the last `overlap_chars` characters of the previous chunk's text (so char_start of N+1 < char_end of N — satisfies test_chunk_overlap_nonzero). If a single block exceeds target_chars, hard-split it at `target_chars` boundaries (still applying overlap).
    - Each emitted Chunk gets: `content` (the packed text), `ordinal` (0-based running index), `section` (the section active at the chunk's start), `char_start`/`char_end` (byte/char offsets into the original `text`).
    - Provide defaults via a small `pub fn chunk_default(text: &str) -> Vec<Chunk>` calling `chunk_text(text, 512, 64)` for callers in Plan 03-03.
