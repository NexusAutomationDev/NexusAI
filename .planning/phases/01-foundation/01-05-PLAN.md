<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner → GitHub Secrets | Signing secrets must never appear in logs |
| Developer machine → git repo | Private key must never be committed |
| Updater endpoint → app | Signed update packages verified against pubkey |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05-01 | Information Disclosure | TAURI_SIGNING_PRIVATE_KEY in CI logs | mitigate | Private key is stored in `secrets.TAURI_SIGNING_PRIVATE_KEY` — GitHub Actions masks secret values in logs. Never echo or print the key in any workflow step. |
| T-01-05-02 | Tampering | YAML injection via tag name in release workflow | mitigate | `tagName: github.ref_name` is a trusted GitHub context variable, not user input. For production, pin action versions to SHA digests. |
| T-01-05-03 | Spoofing | Unsigned app installs without user consent | accept | Phase 1 builds are unsigned (D-11). macOS shows "unidentified developer" warning; Windows shows SmartScreen. Acceptable for internal use during development. Signing activation is documented in SIGNING.md. |
| T-01-05-04 | Information Disclosure | Private key file at ~/.tauri/nexusai.key | mitigate | SIGNING.md §Backup procedure requires key to be stored in a password manager immediately after generation. The file MUST NOT be committed to git. nexusai.key is added to .gitignore in this plan. |
| T-01-05-05 | Tampering | Malicious update via hijacked endpoint | mitigate | Update packages are verified against the pubkey in tauri.conf.json before installation. Even if the endpoint URL is compromised, packages not signed with the private key are rejected by the Tauri updater. |
</threat_model>
