## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-icon gaps, inline badge padding |
| sm | 8px | Within-component spacing (button padding, list item gaps) |
| md | 16px | Default element spacing, settings field gaps |
| lg | 24px | Section padding, sidebar vertical rhythm |
| xl | 32px | Settings panel content padding |
| 2xl | 48px | Major section breaks within settings page |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- **Sidebar icon touch target:** 40px height (sidebar icons must be easily clickable;
  40px is the accessible minimum for desktop pointer interaction — aligns to 8-point grid
  as 5 × 8px)
- **Sidebar width (icon-only):** 48px fixed — icon (20px) centered in 48px column with
  4px horizontal padding each side

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Settings field labels, descriptions, tooltip text |
| Label | 12px | 500 | 1.4 | Badge text, section sub-headers, status indicators |
| Heading | 16px | 600 | 1.3 | Settings section titles (API Keys, Models, Appearance) |
| Display | 20px | 600 | 1.2 | Settings page title ("Settings") |

**Font scale selector (FOUND-03 D-07):**
Three levels controlled via `--font-scale` CSS variable on `<html>`:

| Scale option | `--font-scale` | Effective body size |
|---|---|---|
| Small (sm) | 0.875 | 12.25px |
| Medium (md) | 1 (default) | 14px |
| Large (lg) | 1.125 | 15.75px |

All `rem`-based values cascade from the font scale variable. The four type roles above
are declared at the `md` scale baseline.

## Color

Dark mode is the default and primary theme. Light mode is a toggle (FOUND-03 / D-07).
shadcn/ui CSS variable tokens drive both modes — toggle by adding/removing `.dark`
class on `<html>`.

| Role | Dark mode value | Light mode value | Usage |
|------|-----------------|------------------|-------|
| Dominant (60%) | `hsl(224 71.4% 4.1%)` (near-black) | `hsl(0 0% 100%)` (white) | App background, main content area |
| Secondary (30%) | `hsl(215 27.9% 16.9%)` (dark slate) | `hsl(210 40% 96.1%)` (light gray) | Sidebar column, settings sub-nav panel, cards |
| Accent (10%) | violet — `hsl(263.4 70% 50.4%)` | violet — `hsl(263.4 70% 50.4%)` | See reserved list below |
| Destructive | `hsl(0 72.2% 50.6%)` (red) | `hsl(0 72.2% 50.6%)` (red) | Delete API key action only |

**Accent reserved for (explicit list — all other interactive elements use muted/secondary):**
1. Active/selected sidebar icon highlight ring
2. Focus ring on interactive form elements (inputs, selects)
3. Toggle "on" state (theme toggle, font-scale segment when active)
4. Accent color picker swatch — the selected swatch border/ring indicator

**Accent color palette (D-08) — 5 options, predefined only:**

| Name | CSS attribute value | HSL |
|------|---------------------|-----|
| Violet (default) | `data-accent="violet"` | `hsl(263.4 70% 50.4%)` |
| Blue | `data-accent="blue"` | `hsl(221.2 83.2% 53.3%)` |
| Green | `data-accent="green"` | `hsl(142.1 76.2% 36.3%)` |
| Orange | `data-accent="orange"` | `hsl(24.6 95% 53.1%)` |
| Red | `data-accent="red"` | `hsl(0 72.2% 50.6%)` |

Applied via `document.documentElement.setAttribute('data-accent', value)`. CSS
`@layer` overrides the shadcn `--primary` token per `[data-accent]` selector.

All 5 accent colors must meet WCAG AA contrast (4.5:1) against both dark and light
dominant surfaces. Executor must verify contrast ratios at implementation time.

## Layout Contract

### App Shell (root layout)

```
┌─────────────────────────────────────────────┐
│ 48px sidebar │       Main content area       │
│  (icon-only) │       (flex-grow, 1fr)        │
│              │                               │
│  [icon]      │                               │
│  [icon]      │                               │
│  [icon]      │  (module content / settings)  │
│  [icon]      │                               │
│              │                               │
│  [settings]  │                               │
└─────────────────────────────────────────────┘
```

- Sidebar: `w-12` (48px), `h-screen`, `fixed left-0 top-0`, `flex flex-col`
- Module icons: top section, stacked vertically, 40px touch target each
- Settings gear icon: bottom of sidebar column (`mt-auto`)
- Main content: `ml-12 h-screen overflow-auto`

### Sidebar Icon States

| State | Visual treatment |
|-------|-----------------|
| Active module | Accent color highlight ring (2px, `ring-2 ring-accent`) |
| Hover | `bg-secondary` subtle background |
| Disabled (stub) | `opacity-40`, Tooltip "Em breve" on hover/click |
| Default | No background, muted icon color |

### Settings Page Layout

```
┌──────────────────────────────────────────────┐
│ 48px │  160px sub-nav  │  Main settings panel │
│ side │  ─────────────  │  ───────────────────  │
│ bar  │  API Keys       │  [section content]   │
│      │  Models         │                      │
│      │  Appearance     │                      │
└──────────────────────────────────────────────┘
```

- Settings sub-nav: `w-40` (160px), left side of settings content area
- Settings panel: `flex-1`, `px-8 py-6`, scrollable
- Sub-nav item height: 40px, `text-sm font-medium`

### API Key Field Pattern (D-06)

```
Provider label
[  *** *** *** ***  ] [Editar]   ● Configurado
```

- Input: `type="password"`, `disabled` by default when key is set, `w-72`
- Editar button: clears the field value and enables it for re-entry
- Badge: `variant="outline"`, green text `text-green-500` when configured,
  red text `text-red-500` when not configured
- No "reveal" button — keys must never appear as plaintext in the UI (FOUND-01 security contract)

## Accessibility Requirements

- All interactive elements must be keyboard-navigable (Tab order follows visual layout)
- Sidebar icon-only buttons must have `aria-label` with the module name
- Tooltip on stub icons: triggered on both focus and hover (`delayDuration={0}`)
- Color alone must not convey configured/not-configured state — Badge uses both color
  AND text label
- Focus ring must be visible in both dark and light modes (accent color at 4.5:1 minimum contrast)
- Font scale selector must use `aria-pressed` or equivalent for active state
