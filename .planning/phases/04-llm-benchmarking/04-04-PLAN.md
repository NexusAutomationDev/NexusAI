<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verificação humana do fluxo completo de benchmark</name>
  <what-built>
    Módulo completo de LLM Benchmarking:
    - Rota /benchmark com tabs "Nova sessão" e "Histórico"
    - Estado pré-run: textarea de prompt + 2 seletores de modelo (D-02) + botão "Iniciar benchmark"
    - Estado running: N colunas com ResizablePanelGroup (D-01) — redimensionáveis e colapsáveis
    - Cada coluna streams independentemente via Channel API (BENCH-01)
    - Scoring bar sticky (D-03) auto-habilita quando todas as colunas terminam (BENCH-02)
    - Histórico tab com tabela (D-04): prompt truncado, badges de modelos, badge de vencedor, data relativa PT-BR
    - Ícone Benchmark na sidebar entre KB e Gmail
  </what-built>
  <how-to-verify>
    Iniciar o app com `pnpm tauri dev` e seguir os passos em ordem:

    **1. Sidebar e navegação:**
    - Confirmar que o ícone Benchmark (BarChart2) aparece na sidebar entre KB e Gmail
    - Clicar no ícone — deve navegar para /benchmark
    - Confirmar tab "Nova sessão" ativa e tab "Histórico" disponível

    **2. Estado pré-run (BENCH-01 pre-requisito):**
    - Confirmar que 2 seletores de modelo aparecem pré-preenchidos com defaults de Settings
    - Digitar um prompt no textarea
    - Adicionar um terceiro modelo via "+ Modelo" — confirmar que aparece (máx 4)
    - Remover uma coluna via "×" — confirmar que fica mínimo 2 (botão × desativa com 2 colunas)
    - Botão "Iniciar benchmark" deve ficar desabilitado sem prompt

    **3. Fluxo de streaming paralelo (BENCH-01):**
    - Com 2+ modelos selecionados e um prompt, clicar "Iniciar benchmark"
    - Confirmar que N colunas aparecem lado a lado com ResizablePanelGroup
    - Confirmar que tokens chegam INDEPENDENTEMENTE em cada coluna (cada coluna tem seu próprio streaming indicator Loader2)
    - Confirmar que colunas são redimensionáveis arrastando o handle central
    - Colapsar uma coluna arrastando até menos de 15% — deve colapsar para ~4% mostrando apenas o header
    - Parar uma coluna com o botão ■ — apenas aquela coluna para, as outras continuam

    **4. Scoring bar (BENCH-02):**
    - Confirmar que scoring bar aparece desabilitada (opacity-50) durante streaming
    - Quando TODAS as colunas terminam (done/error), confirmar que scoring bar habilita automaticamente
    - Clicar no nome de um modelo vencedor — confirmar que o ToggleGroupItem fica selecionado (bg-primary)
    - Confirmar que toast "Avaliação registrada." aparece
    - Confirmar que scoring bar fica disabled após scoring (evita re-score)

    **5. Histórico (BENCH-02):**
    - Clicar na tab "Histórico"
    - Confirmar que a sessão recém-avaliada aparece na tabela
    - Confirmar que o badge de vencedor está correto (nome do modelo selecionado, variante "default")
    - Confirmar que a data aparece em formato relativo PT-BR (ex: "há alguns segundos")
    - Hover no prompt truncado — tooltip deve mostrar o texto completo
    - Hover na data relativa — tooltip deve mostrar timestamp absoluto

    **6. Fluxo completo de empate:**
    - Iniciar nova sessão, deixar streaming completar
    - Selecionar "Empate" no ToggleGroup
    - Verificar no Histórico que badge mostra "Empate" com variant="secondary"

    **7. Sessão não avaliada:**
    - Iniciar nova sessão, deixar completar, NÃO fazer scoring
    - Verificar no Histórico (pode precisar recarregar o app) que badge mostra "Não avaliado"

    **Problemas comuns a verificar:**
    - Tokens de colunas diferentes NÃO devem aparecer na coluna errada (isolamento de Channel)
    - Scoring bar NÃO deve habilitar enquanto qualquer coluna ainda está streaming
    - Não deve haver rerenders excessivos ou travamentos durante streaming
  </how-to-verify>

    Ou descreva os problemas encontrados para que sejam corrigidos antes de fechar a fase.
